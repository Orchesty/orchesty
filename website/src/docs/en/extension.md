The concept of PIPES is to make everything straight. It is quite an open tool and most importantly a tool for developers. PIPES allows building custom extensions of more levels:

- We can build new connectors for services in [PIPES Appstore](/docs/cs/pipes-appstore). Custom connectors can be useful if we need to call endpoint which is not included natively in PIPES.
- We can extend [PIPES Appstore](/docs/cs/pipes-appstore) with our own applications. 
- We can build new elements to integrate into our processes. 

The PIPES basis allows building custom tools. Extensions are reusable and can be built in every programming language for which is prepared PIPES [SDK](/docs/cs/sdk). The principle is very simple. You just need to build a service with custom extensions with [SDK](/docs/cs/sdk) and register it into [PIPES Admin](/docs/cs/admin) as [Service for direct integration](/docs/cs/integration).
